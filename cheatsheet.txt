TIPS
====

     Search in manual pages names and descriptions, or as fulltext:

          man -k wifi
          man -K wifi_fulltext

     Kill:

          kill 12345
          kill -9 12345
          pkill Xorg

     Disk usage:

          du -sh /*

     Search:

          find /etc | grep efi
          grep -R "text_inside_file" /etc   2> /dev/null

     Ignore errors:

          find 2> /dev/null

     Suspend process to background, run process on background, move to fg:                          &
          
          Ctrl+Z
          bg
          fg

     Run process on background as daemon service:

          daemon -f -o output.log some_command


LIVE CD RECOVERY
================


     Tips
     ----

     Re-mount the root filesystem for read-write when encountering "read-only file system" error:

          mount -u -o rw /


     Change boot environment
     -----------------------

     1. Import the ZFS partition via "zpool import".

     2. Get active boot environment

          zpool get bootfs zroot

     3. Set a different boot environment:

          zpool set bootfs=zroot/ROOT/before-update-to-15-0 zroot
          reboot 


NETWORK CONNECTION
==================


     Wireless Wifi via DHCP
     ----------------------

     1. List APs:

          ifconfig wlan0 scan


     2. Connect:

          wpa_passphrase "SSID" "PASSWORD" > /etc/wpa_supplicant.conf
          ifconfig wlan0 up
          wpa_supplicant -B -i wlan0 -c /etc/wpa_supplicant.conf
          dhclient wlan0


     LAN connection via DHCP
     -----------------------

     Just find the LAN interface (usualy em0) and connect:

          ifconfig
          dhclient em9


UPGRADE FREEBSD
===============
     
     1. Apply security patches:

          freebsd-update fetch
          freebsd-update install

     2. First step to upgrade:

          freebsd-update -r 13.2-RELEASE upgrade

     3. Follow the instllator, it will want to type several:

          freebsd-update install
          reboot
          freebsd-update install

     4. Update also Ports:

          pkg-static upgrade -f

     5. OPTIONAL: Rebuild all aps installed from ports:

          portmaster -af

     6. At the end:

          freebsd-update install

MOUNT FILESYSTEMS
=================


     Mount encrypted ZFS partition:
     ------------------------------

     Unlock disk, list zpool and mount:

          geli attach /dev/da0p4
          zpool import
          zpool import -o altroot=/mnt POOLNAME


     Mount UNKNOWN disk partition
     ----------------------------

     Detect filesystem, mount and unmount:

          fstyp /dev/da0p1
          mount -t ufs /dev/da0p1 /mnt/disk
          umount /mnt/disk


DISK PARTITIONING
=================


     Create partition scheme on an external disk
     -------------------------------------------

     Exact example of default FreeBSD partitioning scheme during system installation.

     1. Show disks and partitions:

          gpart show
          geom disk list

     2. Destroy and then create new GPT partition table:

          gpart destroy -F da0
          gpart create -s gpt da0

     3. Create EFI, FreeBSD boot, swap:

          gpart add -t efi -s 260M da0
          gpart add -t freebsd-boot -s 512K da0
          gpart add -t freebsd-swap -s 4G da0

     5. Create ZFS partition *without* ``-s`` so it will stretch to the end of the disk:

          gpart add -t freebsd-zfs da0

     6. Format EFI and ZFS partition, boot and swap don't have to be formatted:

          newfs_msdos -F 32 -c 1 /dev/da0p1
          zpool create mypool /dev/da0p4

     8. Check result:

          gpart show da0
          zpool status
          zfs list


BACKUP AND RESTORE WITH ZFS ROOT PARTITION
==========================================


     Backup and restore partition table, /EFI and /boot partitions
     -------------------------------------------------------------

     1. Backup GPT, /efi and /boot:

          gpart backup da0 >   /mnt/backup_gpart_backup_partition_scheme.backup
          dd if=/dev/da0p2 of=/mnt/backup_freebsd_boot.img bs=512 status=progress
          tar -C /mnt/efi -cf  /mnt/backup_efi_partition.tar .

     2. Restore GPT, /efi and /boot:

          gpart restore da0 < /mnt/backup_gpart_backup_partition_scheme.backup
          tar -C /mnt/efi -xf /mnt/efi-partition.tar
          dd if=/mnt/da0p2-freebsd-boot.img of=/dev/da0p2 bs=512


     Backup and recover root partition with ZFS
     ------------------------------------------

     1. Create a boot environment from the current root system and verify:
     
          bectl create default-2026-06-17T12:00     
          bectl list
     
     2. Do the FreeBSD upgrade:
     
          freebsd-update -r 15.0-RELEASE upgrade
          freebsd-update install
          reboot
          freebsd-update install
          << FATAL_ERROR
     
     3. If the upgrade fails, boot the previous environment:
     
          bectl activate default-2026-06-17T12:00
          reboot

     NOTE: You can see that bectl created new root:

          zfs list -t snapshot          
     

     Backup and recover root partition from external drive
     -----------------------------------------------------

     1. Send snapshot to externsal disk:

          zfs send zroot/ROOT/default@BEFORE_UPGRADE_TO_V15.0 > /mnt/zroot--ROOT--default@BEFORE_UPGRADE_TO_V15.0.zfs

     2. Receive snasphot from external disk to new default:

          zfs receive zroot/ROOT/new_default < /mnt/zroot--ROOT--default@BEFORE_UPGRADE_TO_V15.0.zfs

     3. Activate new Boot environment for next reboot:

          bectl activate new_default
          reboot
